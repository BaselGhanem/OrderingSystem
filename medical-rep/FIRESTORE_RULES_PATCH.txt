Add these collection matches to your existing Firestore rules if the new medical-rep pages show "Missing or insufficient permissions".

Important: keep your current orders/pharmacies rules unchanged. The portal reads orders and pharmacies, and writes only its own setup collections from medical-rep/admin.html.

match /medicalReps/{docId} {
  allow read: if true;
  allow write: if true;
}

match /medicalRepAreaRules/{docId} {
  allow read: if true;
  allow write: if true;
}

match /medicalRepOtherShares/{docId} {
  allow read: if true;
  allow write: if true;
}

match /medicalRepTargets/{docId} {
  allow read: if true;
  allow write: if true;
}
